# 6. Teoría: Tipos de Licencias de programas vs Sistemas operativos

## 1. Clasificación de sistemas operativos según la licencia

Los sistemas operativos (SO) se caracterizan por la **licencia de uso del usuario final**. Esa licencia corresponde al contrato de privacidad, términos de uso y cualquier otra legislación que acompaña al software.

- **Licencia OEM (Original Equipment Manufacturer):**
   Preinstalada en equipos originales por fabricantes como Dell, HP, Lenovo, etc. Puede ser válida solo para ese hardware.
- **Licencia Retail:**
   Se compra en tienda física o digital. Permite instalar el sistema en distintos equipos (aunque no simultáneamente) y se activa con una clave.
- **Licencia por volumen:**
   Pensada para instituciones educativas, administraciones públicas o grandes empresas. Permite instalar el mismo SO en muchos equipos gestionados bajo un contrato.
- **Licencia MSDN (Microsoft Developer Network):**
   Diseñada para desarrolladores. Permite acceder a un amplio catálogo de software y versiones de sistemas para pruebas y desarrollo.

**Nota importante**: El tipo de licencia afecta a la legalidad del uso, al soporte técnico recibido y, en ocasiones, a la posibilidad de actualizar el software.

## 2. Arquitectura de un sistema operativo

Un sistema operativo tiene una estructura lógica por capas que controla, organiza y gestiona los recursos físicos y lógicos de un sistema informático. Sus principales componentes son:

- **Kernel (núcleo):**
   Gestiona directamente el hardware: CPU, RAM, almacenamiento, dispositivos de entrada/salida. Proporciona abstracción para que el software lo utilice de forma segura, es decir, actúa como intermediario entre el **hardware** y el **espacio de usuario** (programas, servicios, shells, etc.).
- **Controladores (drivers):**
   Software que permite que el SO se comunique con dispositivos concretos (impresoras, tarjetas gráficas, etc.).
- **Servicios del sistema:**
   Programas que se ejecutan en segundo plano (demonios en Linux) y que gestionan usuarios, accesos, red, archivos y más.
- **Aplicaciones:**
   Programas que resuelven tareas concretas para el usuario.

Arquitectura representada en capas:
 Aplicaciones → Servicios → Controladores → Kernel

## 3. Funciones de un sistema operativo

El sistema operativo debe:

- Gestionar usuarios.

  El SO permite crear, modificar y eliminar cuentas de usuario. Define qué puede hacer cada uno: administrador (*root* en Linux, *Administrador* en Windows) o usuarios normales con permisos limitados. Controla permisos de archivos y directorios (lectura, escritura, ejecución).

  -En Linux puedes tener un usuario estudiante sin privilegios, y otro usuario profesor con permisos para instalar software.

- Gestionar dispositivos de E/S.

  Se encarga de que el software pueda comunicarse con el hardware a través de drivers (controladores). lSO abstrae la complejidad del hardware: el programa no necesita saber los detalles eléctricos de un teclado o de un disco, solo usa funciones estándar. 

  -Cuando guardas un archivo en un USB, el SO coordina la petición, llama al driver del USB y envía los datos al dispositivo.

- Gestionar procesos.

  Ya vimos que un proceso (en el punto de teoría anterior) es un programa en ejecución, con su memoria, variables y contexto. El SO decide qué procesos se ejecutan, en qué orden y durante cuánto tiempo , esto lo hace el planificador (scheduler). El scheduler de Ubuntu (y de cualquier Linux moderno) es el Completely Fair Scheduler (CFS). Es un planificador generalista que reparte la CPU de manera justa entre procesos, aunque también existen planificadores en tiempo real. Si hay más procesos que núcleos disponibles, el SO reparte el tiempo entre ellos (multitarea). 

  -Puedes escuchar música en Spotify mientras navegas con Firefox y a la vez compilas un programa en Python → todos son procesos que el SO coordina.

- Gestionar memoria principal.

  La RAM es limitada, y el SO decide qué parte se asigna a cada proceso. Evita que un proceso “pise” la memoria de otro. Si falta memoria, el SO usa memoria secundaria (swap) para simular más RAM. 

  -Si tienes abierto un videojuego y a la vez un navegador con muchas pestañas, el SO asigna RAM a ambos, pero puede “sacar” pestañas a disco (swap) cuando no se usan mucho.

- Gestionar almacenamiento secundario.

  Maneja discos duros, SSDs o cualquier dispositivo donde se almacenen datos de forma persistente, organiza la información en sistemas de archivos (NTFS en Windows, ext4 en Linux, APFS en macOS…) y controla qué usuario o proceso puede leer/escribir en cada archivo.

  -Cuando guardas un documento en Word, el SO traduce esa acción en operaciones concretas sobre el sistema de archivos de tu disco.

- Gestionar comunicaciones y red.

  Permite que los procesos se comuniquen entre sí, ya sea en el mismo ordenador o a través de una red e implementa protocolos de comunicación (TCP/IP, UDP, HTTP…). También administra interfaces de red (Ethernet, Wi-Fi).
  
   -Cuando haces una videollamada por Zoom, el SO se encarga de enviar los datos de tu micrófono y cámara a través de Internet y de recibir los de la otra persona.

Además, cumple dos funciones transversales:

- **Seguridad:** proteger los datos y controlar accesos.
- **Abstracción:** simplificar la complejidad del hardware ofreciendo una interfaz manejable al usuario.

## 4. Interfaces

La **interfaz** es el medio de comunicación entre el usuario y el sistema operativo.
 Puede ser:

- **De texto** (línea de comandos, CLI).
- **Gráfica** (ventanas, iconos, menús, puntero).

La calidad de una interfaz afecta directamente a la usabilidad, la eficiencia y la forma en que el usuario percibe el sistema.
